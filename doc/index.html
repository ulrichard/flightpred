Free Flight Prediction


A system that shall predict good days and locations for cross country free flying such as paragliding by comparing current weather predictions with statistics about past weather predictions and flights from online contests.
Short description
To do so, we collect weather prediction data from the past, and match them with the flights that were uploaded to online competitions. Using these data, we train a self learning system. Training will probably be done per flight area. Then, when we want to predict the upcoming days, we fetch the current weather predictions, feed them to the system and create a ranking of which areas have the best xc potential for the upcoming days.

Input

Weather Data
Weather prediction systems calculate various parameters for points on a grid at various altitudes. These values can be downloaded as grib files. At the beginning we will use data from the GFS model run by the noaa, as it is easily accessible Once we have some results, we might also want to look for data from a high quality meso scale model.
To train our system, we have to extract some meaningful features at some grid positions that are sensible for the flight area for which we want to train. We will have to evaluate how many grid points and altitude (or pressure) steps are feasible.
These features could be:
  • year
  • day of year
  • per selected grid point and altitude step:
    • temperature difference to the next step (more important for thermals than the bare temperature [to be prooved])
    • wind speed
    • pressure (or altitude if the grid is pressure based)
    • humidity (or skew)
It looks like the "weather database system" could help in handling the grib data, so we will give it a try.
Actual Flights from the online competitions
The measures that we want to predict have to be known for the past. So, we will calculate these measures from the flights on the online competitions such as xcontest.org or onlinecontest.org possibly also from Leonardo or others.
The measures will be calculated (for the past) and predicted (for the future) per flight area and could include:
  • maximum distance
  • number of flights
  • average distance
  • number of pilots having the best flight of the season.

The self learning system

At first, we will use a support vector machine possibly from torch but may later also experiment with neural networks or with voting of both systems. The engine shall be pluggable to easily experiment with different systems.
Depending if the machine can handle multiple outputs, we willl generate one training sample per day or as many as we have figures to predict. Each training sample consists of all the extracted features from the weather predictions and the desired result.

Platform

The main development is in C++ on linux, specifically debian based systems such as ubuntu or voyage. The code is open so it should be easy to port to other platforms such as mac os x or Windows, but the actual porting will come later, if at all.
